# LendIQ — GenAI Copilot for a Securities Lending Desk
 
A production-style GenAI application that answers securities lending desk questions three ways: guarded NL→SQL over the lending book, clause-cited RAG over GMSLA agreements, and an automated locate workflow where a deterministic rules engine decides and the LLM only drafts the response.


The Problem
A securities lending desk handles thousands of locate requests a day, dozens of recalls, margin calls, and constant client queries like "why did my borrow fee on GME jump?" or "what does our GMSLA with Counterparty X say about collateral substitution?" Traders spend a large share of their time answering routine questions by digging through the loan book, availability feeds, and 80-page legal agreements.

LendIQ is an AI copilot that answers desk questions instantly, grounded in actual data and documents, with zero hallucinated numbers — because a wrong rate quoted to a client is a compliance incident.
Capabilities
Quantitative Q&A against the lending book — "What's our utilization on NVDA?", "List open loans to Counterparty ABC above $10M with negative rebates" — via natural-language-to-SQL with a deterministic safety validator.
Contractual Q&A against MSLA/GMSLA agreements — "What is the recall notice period in our agreement with Meridian Capital?" — via RAG with clause-level citations.
Locate handling end-to-end — parse a free-text locate request, check availability, apply desk rules (HTB list, client tier), and draft a response with quantity and indicative rate. High-value locates require human approval.
Morning briefing — automated summary of overnight recalls, specials whose fees moved >100 bps, pending margin calls, and top utilization changes.
Safe refusal — declines out-of-scope questions, never invents numbers, and cites the source (SQL query or document clause) for every fact.

Architecture (summary)
User → FastAPI → LangGraph Router → ┬→ SQL Analyst Agent  → SQLite (read-only)

                                    ├→ Contract RAG Agent → ChromaDB (GMSLA clauses)

                                    ├→ Locate Workflow    → Rules engine (pure Python) + LLM drafting

                                    ├→ Briefing Generator → Canned queries + summarization

                                    └→ Safe Reject

Core principle: the LLM orchestrates and explains; deterministic systems compute. Full architecture discussion, HLD and LLD in /docs.
Tech Stack
Python 3.11 · LangGraph · Anthropic API · ChromaDB · sentence-transformers · SQLite → PostgreSQL · FastAPI · Streamlit · pytest · Docker

All data is synthetic — generated by src/datagen/ to model a realistic lending book (GC vs specials, cash/non-cash collateral, rebate ≈ benchmark − fee, recalls, margin calls). Fully reproducible: no proprietary data anywhere in this repo.
